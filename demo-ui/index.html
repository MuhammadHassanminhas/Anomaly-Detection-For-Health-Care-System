Phase 4 · Step 7

Every check a machine drafts still needs someone to say yes.

This is the review gate: the one place a check moves from "the system suggested this" to "a person is willing to act on this." Nothing reaches a clinic's queue without going through it.

The commands below are real — this is the actual command-line tool the team uses. The output is a recording from a real review session, played back here so it can be shown without a live database connection in the room.

What this earns next

      
The review gate isn't the finish line — it's what makes everything after it trustworthy. Once a check is active, here's what starts using it.

      

        

          
Phase 5

          
Findings get a voice

          
An approved check that fires produces a finding. Phase 5 turns that into a plain sentence for staff — with every number filled in by code, never guessed by the model.

        

        

          
Phases 6 & 7

          
The library keeps itself honest

          
Staff feedback on dismissed findings feeds a precision score per check, per practice. Checks that cry wolf too often get quietly demoted — and new candidate checks flow back through this same review gate.

        

        

          
Phases 9 & 10

          
A real screen for real people

          
Everything shown on this page as a terminal command becomes an API and a proper triage UI — a queue, evidence, one-click dismiss-with-reason. This page is a preview of that workflow, not a replacement for it.
