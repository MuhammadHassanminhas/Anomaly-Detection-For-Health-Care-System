# Phase 0 — Environment & access verification

## Objective

Prove scripted, audited, read-only access to `INDICI_BI_Full`; enumerate the view surface the account can actually see and reconcile it against the 56 export names (D-001); capture SQL Server version, per-view row counts, and watermark-column candidates into a machine-readable environment report.

## Preconditions

- D-002 delivered: host/port, auth method, credentials — consumed **only** from environment variables (`CDSS_SOURCE_HOST`, `CDSS_SOURCE_PORT`, `CDSS_SOURCE_DB`, `CDSS_SOURCE_USER`, `CDSS_SOURCE_PASSWORD`, `CDSS_SOURCE_ENCRYPT`); a gitignored `.env` + committed `.env.example`.
- Python 3.12+, ODBC Driver 18, `pyodbc` installable on this machine (D-014).

## Steps (one deliverable each; TDD — failing test first for every behavior)

1. **Repo scaffolding.** `pyproject.toml` (ruff, mypy strict, pytest), `src/cdss/` package skeleton, `scripts/verify_env.ps1` stub, CI-runnable `scripts/check.ps1` (lint + typecheck + tests).
   *Deliverable:* `scripts/check.ps1` runs green on the empty skeleton.
2. **Config loader.** `cdss.config` reads the `CDSS_SOURCE_*` variables, fails fast with a clear message listing missing names; never logs values.
   *Deliverable:* unit tests for present/missing/partial env; a log-scrubbing test proving no secret value appears in any log line.
3. **Audited source access layer (v0).** `cdss.source`: opens a read-only connection; every statement goes through one choke point that (a) appends a JSONL audit event to `artifacts/audit/source-audit-YYYYMMDD.jsonl` — statement, params, UTC timestamp, duration_ms, rows_returned, component (D-016) — and (b) rejects any statement that is not a single `SELECT` (sqlglot parse; DML/DDL/multi-statement refused) against an allowlist of: `INFORMATION_SCHEMA` / `sys` catalog views (D-015) or the enumerated view surface once known.
   *Deliverable:* unit tests — DML/DDL/multi-statement/base-table-name injection all refused; every accepted statement produces exactly one audit line (tested against a fake connection; no live DB needed).
4. **Connection smoke + version capture.** `SELECT @@VERSION`, `SERVERPROPERTY('ProductVersion'/'Edition'/'EngineEdition')`, confirm `DB_NAME() = 'INDICI_BI_Full'`; record driver/TLS details.
   *Deliverable:* live smoke passes; version facts land in the report structure.
5. **Surface enumeration.** Query `INFORMATION_SCHEMA.VIEWS`, `INFORMATION_SCHEMA.TABLES`, and effective permissions to list every object the account can `SELECT` from, with object type (view vs base table) per `sys.objects`.
   *Deliverable:* enumerated surface in the report, each object tagged view/table/other.
6. **Reconciliation against the export (D-001).** Match the 56 export names to the enumerated surface: found-as-view / found-as-table / missing / extra objects. No judgment call is made in code — the raw reconciliation table is the deliverable for the product owner.
   *Deliverable:* reconciliation section of the report; discrepancy list.
7. **Row counts + watermark candidates.** For each accessible in-scope object: exact `COUNT(*)` (or partition-stat count if COUNT exceeds a timeout, then explicitly marked approximate — never presented as exact), presence/type of `InsertedAt`/`UpdatedAt`, and `MIN`/`MAX` of each watermark candidate. Per-query timeout + cost (duration) recorded per F10.
   *Deliverable:* per-object stats in the report; the 12 known watermark-less objects confirmed or corrected from live schema.
8. **Environment report artifact.** `scripts/verify_env.ps1` (wrapping `python -m cdss.verify_env`) runs steps 4–7 end-to-end and writes `artifacts/env-report.json` (machine-readable, schema-validated) + `artifacts/env-report.md` (human-readable). Idempotent; re-run overwrites.
   *Deliverable:* both artifacts generated by one command from a clean checkout.

## Exit criteria (all objectively verifiable)

1. `scripts/check.ps1` exits 0 (lint, mypy strict, all unit tests).
2. `scripts/verify_env.ps1` exits 0 against the live source and writes both report artifacts.
3. `artifacts/env-report.json` validates against `src/cdss/schemas/env-report.schema.json`.
4. Audit JSONL exists and contains one line per live statement executed; line count matches the executor's statement count.
5. SQL-guard tests prove refusal of DML, DDL, multi-statement, and non-allowlisted objects.
6. Report contains: SQL Server version/edition, reconciliation table for all 56 names, row counts, watermark candidates.
7. **Product-owner sign-off on D-001** recorded in `DECISIONS.md` (status → DECIDED) based on the reconciliation table. *This is the gate; the phase cannot close without it.*

## Risks / dependencies / open questions

- **D-002 (blocker):** cannot start live steps 4–8 without credentials; steps 1–3 proceed offline.
- **Enumeration may reveal base tables are visible.** We still only read metadata about them (D-015); the reconciliation report is where this surfaces. If the 56 names resolve to base tables and no equivalent views exist, that is a D-001 outcome for you to rule on — possibly the first `ASK-NNN`.
- **`COUNT(*)` cost on large objects** (e.g., `dbo.Measurements`, `dbo.Appointments`): mitigated by per-query timeout + explicit approximate fallback (step 7).
- **ODBC Driver 18 availability** on this machine: checked in step 1; install is a documented manual prerequisite (allowed — one-time setup, not a pipeline step).
- No new `DECISIONS.md` entries required by this spec.
